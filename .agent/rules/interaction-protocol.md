# AIエージェント対話・実行プロトコル

## 1. Directive（指示）と Inquiry（問い合わせ）の厳格な区別
- **Inquiry (問い合わせ/報告)**: 「確認してください」「レビューしてください」「〇〇しました」「〇〇案を採用します」など。
  - **制約**: この場合、エージェントは**ファイルの作成・編集・削除を一切行ってはならない**。読み取り専用ツール（`read_file`, `grep_search` 等）による分析と、テキストでの回答のみを許可する。
- **Directive (実行指示)**: 「実装してください」「修正を適用してください」「フェーズ2に着手してください」「コミットしてください」など。
  - **制約**: 明確な実行命令がある場合のみ、ファイル操作ツールを許可する。

## 2. マイルストーンごとの「停止・待機（Stop & Wait）」の義務化
- 計画書（`docs/plans/*.md`）が存在する場合でも、現在のタスクまたは指示された箇所までのマイルストーンが完了した時点で、**必ず作業を停止し、ユーザーの次の指示を待たなければならない**。
- 「次は何をしますか？」という確認を行わずに、独断で次のマイルストーンの実装を開始することを固く禁ずる。

## 3. 暗黙の指示の排除
- 「〇〇案で行きます」という方針提示は、計画書の修正や実装への着手許可ではない。必ず「計画を更新してください」等の具体的な命令があるまで待機すること。
