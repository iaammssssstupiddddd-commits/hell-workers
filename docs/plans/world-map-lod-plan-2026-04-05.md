## 1. 目的

- **課題**: 地形は **約 10,000 エンティティ**（`spawn_map` で 1 セル 1 `Mesh3d`）。シェーダは `terrain_id_map` 近傍サンプル・境界ブレンド・マクロノイズ等を含み、**ズームアウト時もドロー数とピクセルシェーダ負荷が線形に残る**。
- **到達状態**: カメラの **見え方スケール（ortho scale または画面内タイルサイズ）** に応じて、**ドロー数・シェーダ複雑度・境界表現**を段階的に落とし、60fps 運用の余裕を確保する。
- **非目的**: ゲームロジックの `WorldMap` / グリッド解像度の変更、経路探索コストの変更（LOD は **描画専用**）。

---

## 2. 現状の実装前提（確定事項）

| 項目 | 実態 |
| --- | --- |
| グリッド | `MAP_WIDTH` × `MAP_HEIGHT` = **100 × 100**（`docs/world_layout.md`） |
| タイルサイズ | `TILE_SIZE` = **32.0**（ワールド単位） |
| スポーン | `crates/bevy_app/src/world/map/spawn.rs` — セルごとに `Mesh3d(tile_mesh)` + `TerrainSurfaceMaterial` |
| ズーム | `PanCamera` が **MainCamera（2D）** の `transform.scale` を更新 → `sync_camera3d_system` が **3D `OrthographicProjection.scale`** にコピー（`camera_sync.rs`） |
| 既存メモ | MS-3-6: **「地形チャンク化・LOD は別マイルストーン」** |

**LOD の駆動変数**: 実質的には **`cam2d.scale.x`（= ortho scale）** または、これから導出する **「画面内に収まるタイル数」** / **1 タイルの画面ピクセル幅** が安定した閾値になる。

## 3. LOD 方針の全体像

### 3.1 レイヤー分割（推奨）

ワールドマップの描画は次のレイヤーに分けて LOD を設計する。

| レイヤー | 内容 | LOD の主対象 |
| --- | --- | --- |
| **A. ベース地形** | `TerrainSurfaceMaterial` + タイルメッシュ | **本計画の主戦場**（ドロー数・シェーダ） |
| **B. 境界オーバーレイ** | 曲線境界メッシュ等（`curved-tile-boundaries` 系） | 遠景では **非表示または単純化** |
| **C. 2D 前景** | 木・資源・キャラ等スプライト | 既存のキャラ/オブジェクト LOD 方針と整合（別タスク可） |

### 3.2 LOD レベル案（ワールド地形）

命名は既存の **LOD0 / LOD1 / LOD2**（`docs/art-style-criteria.md` の建物向け）と揃え、**用途**だけ地形用に定義する。

| レベル | 想定ズーム | 描画戦略（案） | 品質目標 |
| --- | --- | --- | --- |
| **LOD0** | 近〜中（プレイの主帯域） | **現状どおり** 1 タイル 1 メッシュ + フル `TerrainSurfaceMaterial` | 境界ブレンド・ノイズ・river 表現を維持 |
| **LOD1** | 中〜遠 | **(1)** チャンク統合メッシュ、または **(2)** 同一メッシュのまま **シェーダ簡略化 variant**（ブレンド幅ゼロ・ノイズオフ等） | 色分けと大まかな地形帯が読める |
| **LOD2** | 最遠（マップ全体俯瞰に近い） | **低解像度単一（または少数）メッシュ** + 単純アルベド／または **事前ベイクしたアルベド 1 枚** | シルエットと川の位置が分かれば足りる |

**閾値の決め方（実装時）**:

1. `ortho.scale` の上下限（PanCamera のズーム範囲）をログまたはデバッグ表示で実測する。
2. 「1 タイルが N ピクセル未満になったら LOD1」など **スクリーン基準**を併用すると、解像度差でブレない。

## 4. 実装オプションの比較

### 4.1 オプション A — 空間チャンク統合（CPU メッシュ or 動的バッチ）

- **内容**: 例: **8×8 または 16×16 セル**を 1 メッシュにし、頂点カラーまたは UV で地形種を渡す。遠景ではチャンク単位のドローに切り替え。
- **利点**: ドロー数が **1/64〜1/256** に近い比率で削減可能。
- **欠点**: `terrain_id_map` や `TerrainChangedEvent` による **部分更新**とチャンク境界の整合が必要。実装・デバッグコストが最大。

### 4.2 オプション B — シェーダー LOD（マテリアル variant / `#[shader_def]` 分岐）

- **内容**: 同じ 10k メッシュでも、**遠景では `#define TERRAIN_LOD1`** のように分岐し、近傍サンプル数・ノイズ・詳細を省略。
- **利点**: メッシュ構造・ECS レイアウトを変えずに **ピクセルコストを下げられる**。
- **欠点**: **ドロー数は 10k のまま**。CPU 側でカメラ距離に応じて `TerrainSurfaceMaterial` のフラグを更新するコスト（全タイル走査の回避策が要る）。

### 4.3 オプション C — GPU インスタンシング / 自動バッチング強化

- **内容**: Bevy のパイプライン特性に依存。**同一メッシュ・同一マテリアル**は既にバッチしやすいが、**10k ドロー**がボトルネックなら明示的 instancing を検討。
- **利点**: 中間案。
- **欠点**: エンジン・バージョンごとの挙動確認が必要（0.18 の推奨パターンをドキュメント化してから）。

### 4.4 オプション D — 遠景用インポスター（RT または低解像度テクスチャ）

- **内容**: 一定間隔で **地形のアルベドを低解像度にレンダリターゲット**し、最遠 LOD で **1 枚ポリゴン**に貼る。
- **利点**: ドロー・シェーダとも最小。
- **欠点**: メモリ・同期・カメラ追従の遅延（パン時のにじみ）対策が必要。

**推奨順（段階導入）**:

1. **B（シェーダー LOD）** — 変更範囲が限定的で検証しやすい。
2. **A（チャンク）** — プロファイルでドロー数が依然として高い場合に本格投入。
3. **D** — 最終手段または「ミニマップ専用」など用途限定で。

## 5. データ・イベントとの整合

| 要因 | 注意 |
| --- | --- |
| `TerrainChangedEvent` → `terrain_id_map` 更新 | LOD1/2 でチャンク化する場合、**該当チャンクの再生成**が必要 |
| `SectionCut` / 矢視 | 地形シェーダの LOD 分岐は **矢視でも破綻しない**こと（MS-3-6 の残チェックと共通） |
| 境界曲線メッシュ | 遠景 LOD では **スキップ**または **簡略ポリライン**でよい（計画 `curved-tile-boundaries` と独立） |

## 6. 成功指標（KPI）

| 指標 | 目安 |
| --- | --- |
| フレーム時間 | 代表マシンで **地形のみ**の GPU 時間または `Render` フェーズが X% 削減（ベースラインはプロファイル取得後に数値化） |
| ドロー呼び出し | 遠景ズームで **10k → 数百〜数千**（チャンクサイズに依存） |
| 視覚 | LOD 切替で **1 フレームに明滅しない**（ヒステリシス付き閾値） |

## 7. マイルストーン案

| 段階 | 内容 |
| --- | --- |
| **M0** | ズーム範囲・`ortho.scale`・1 タイルのピクセルサイズを実測し、LOD 閾値のたたき台を固定 |
| **M1** | シェーダー簡略化（LOD1）+ 全タイル走査を避ける **空間インデックス or チャンク単位のカメラクエリ** |
| **M2** | チャンクメッシュ（LOD1/2）または遠景インポスター |
| **M3** | 境界曲線・建物 LOD との **統合テスト**とドキュメント更新（`docs/architecture.md` または `hw_visual/README.md`） |

## 8. リスクと回避

| リスク | 回避策 |
| --- | --- |
| LOD 切替のポッピング | 閾値に **ヒステリシス**、または数フレーム補間 |
| チャンクと部分更新の不整合 | チャンク ID を `WorldMap` インデックスと明確に対応づけ、イベントは **チャンク粒度**でバッチ |
| 作業範囲の肥大化 | **地形 LOD だけ**を先行し、2D スプライトの間引きは別計画に分離 |
