## 1. 背景と世界観設定 (Lore & Concept)

地獄の王が求める大規模本設施設（「罪人大楼」等）の建設において、空調（気流）と衛生（給排水）は、人間界のような「労働者の快適さ」のためではなく、**「地獄の物理法則と王の建築基準」を満たすための必須インフラ**として存在します。

地獄の本設建築において、部屋とは壁で囲まれた空間ではありません。王の基準では、瘴気を逃がす風路と、罪泥を流す水路が通って初めて、その空間は「使用に耐える苦役施設」として認可されます。

```
┌──────────────────────────────────────────────┐
│                  【本設部屋】                │
│                                              │
│   [風のインフラ]                             │
│   叫びの風穴 💨 ────> 怠惰の瘴気(Miasma)を防止 │
│                                              │
│   [水のインフラ]                             │
│   骨の導水管 💧 ────> 罪泥(Lethe Sludge)を排泥 │
│                                              │
│   ※満たされない部屋は「機能停止・動作不可」  │
└──────────────────────────────────────────────┘
```

### 1.0 王の建築基準：認可済み部屋（Sanctioned Room）
* **世界観**: `Room` 検出で成立した密閉空間は、地獄の行政上はまだ「密閉された罪の箱」に過ぎません。換気と排泥の条件を満たした部屋だけが **認可済み部屋（Sanctioned Room）** として扱われ、本設設備の稼働許可を得ます。
* **ゲーム的意味**: Room 成立は建築の第一段階、インフラ接続は本設利用の第二段階です。仮設設備は現場裁量で動かせますが、王の監査対象になる本設設備は認可済み部屋でなければ動作しません。
* **UI 表現案**: Room overlay に `Unsealed / Room / Sanctioned` のような状態差を持たせ、インフラ不足時は「換気不足」「排泥不足」を理由として表示します。

### 1.1 風のインフラ：怠惰の侵食を防ぐ「風の循環（Screaming Vent）」
* **世界観**: 密閉された部屋（Room）を放置すると、空間に囚われた魂たちの「怠惰の念」が「**怠惰の瘴気（Miasma）**」として物質化し、室内に滞留します。この瘴気は本設の石造建築を腐食させ、機械設備を「怠惰の呪い」で機能停止させてしまいます。王は瘴気に塗れた施設の稼働を認めないため、部屋には必ず風を循環させる設備が義務付けられます。
* **補足設定**: 怠惰の瘴気は単なる悪臭ではなく、部屋の内部時間を遅らせる停滞現象です。換気が足りない部屋では、歯車、弁、炉、祭壇などの動作周期が鈍り、やがて「まだ働き始めなくてよい」という呪いに飲まれて停止します。

### 1.2 水のインフラ：罪泥の堆積を洗い流す「骨の導水路（Ossuary Conduit）」
* **世界観**: 本設の建物や設備が稼働すると、魂たちの活動摩擦により、生前の罪が形となった黒いヘドロ「**罪泥（Lethe Sludge）**」が物理的に堆積します。これを放置すると、設備は目詰まりを起こして機能停止します。そのため、「忘却の川」から水を引いて罪泥を洗い流し、排泥するインフラ配管が必要となります。
* **補足設定**: 忘却の川の水は清潔な水ではなく、設備にこびりついた罪の記憶を剥がす溶媒です。断水した設備は汚れるのではなく、過去の罪を思い出して自壊的に停止します。
* **仮設の免除**: `MudMixer`（泥ミキサー）や `SoulSpa` などの仮設・一時的な設備は、王の建築基準（インフラ要件）の対象外とされ、罪泥の堆積を無視して手動の水搬送で稼働できます。

## 2. 目的（Goals）

- 空調・衛生を人間界の概念から脱却させ、**「怠惰の瘴気（風）」と「罪泥の排泥（水）」という地獄の建築ルール**として導入する。
- `Room` を「囲われた空間」から「王の基準に認可された本設空間」へ段階的に発展させる。
- 密閉された `Room` 内の換気率が基準を満たさない場合、その部屋にある本設設備が**「怠惰の侵食」により稼働不可（機能停止）**になる仕様を実装する。
- 川から本設設備（洗浄盤等）へ水を送る「骨の配管（Ossuary Conduit）」ネットワークを構築し、インフラ未接続（断水）時の設備停止を制御する。

## 3. 新規建築物と世界観表現

### 3.1 風（空調）設備

| 設備名 | 世界観説明 | ゲーム的機能 |
|:---|:---|:---|
| **叫びの風穴**<br>`Screaming Vent` | 埋め込まれた罪人の叫び声を風圧に変換して空気を押し出す、石造りの換気口。壁に埋め込んで設置する。 | 壁用(1x1)。非電化。<br>換気能力【小】。 |
| **溜息煙突**<br>`Sigh Chimney` | 部屋に溜まった怠惰な溜息を屋外へ逃がす縦穴。強制排気ではないが、外周に接している小部屋では安定して機能する。 | 壁/外周用(1x1)。非電化。<br>換気能力【中】。屋外接続が必要。 |
| **黒鞭の送風機**<br>`Scourge Fan` | 鞭を振るうような激しいファン回転により、部屋の怠惰な空気を強制的に排気する鉄製の大型送風機。 | 床置き(1x1)。要電力。<br>換気能力【大】。 |
| **瘴気検査針**<br>`Miasma Gauge` | 部屋の停滞度を測る呪術計器。針が寝るほど部屋そのものが怠けていることを示す。 | 床置き(1x1)。<br>換気状態の可視化。将来のUI補助設備。 |

### 3.2 水（衛生）設備

| 設備名 | 世界観説明 | ゲーム的機能 |
|:---|:---|:---|
| **忘却の川の取水門**<br>`Lethe Intake Gate` | 記憶を消し去る「忘却の川」の黒い水を汲み上げ、インフララインに加圧送水する大型の取水装置。 | 川岸限定(3x2)。<br>要電力。水源供給。 |
| **骨の導水管**<br>`Ossuary Conduit` | 魂の残滓（骨）を繋ぎ合わせて作った中空の配管。インフラとして壁や床の下に重ねて敷設できる。 | 配管(1x1)。素材: 骨。<br>ドラッグ配置対応。 |
| **罪泥の浄化盤**<br>`Sludge Purifier` | 導水管から引いた忘却の川の水で、本設設備や部屋から発生する「罪泥」を自動で洗い流し、排泥する盤。 | 本設設備(1x1)。<br>要配管接続（水）。 |
| **懺悔弁**<br>`Confession Valve` | 導水管の節目に取り付ける弁。開くたびに管内の罪泥が短い懺悔音を立てる。 | 配管制御(1x1)。<br>将来の区画切替・保守用。 |
| **罪泥溜め**<br>`Sludge Basin` | 排出しきれない罪泥を一時的に沈殿させる黒い水槽。あふれると周囲の設備停止リスクが上がる。 | バッファ(2x1)。<br>断水/過負荷時の猶予。 |

## 4. 詳細設計

### 4.0 インフラ責務の切り分け

| 系統 | 判定単位 | 主な失敗状態 | ゲーム上の圧力 |
|:---|:---|:---|:---|
| **風（換気）** | `Room` 単位 | 怠惰の侵食 | 部屋の広さ・密閉度に応じて換気設備を増やす |
| **水（排泥）** | 本設設備または `SludgePurifier` 単位 | 断水・泥詰まり | 川岸から本設区画まで配管網を計画する |

風は「部屋を認可する」ための面管理、水は「設備を稼働し続ける」ためのネットワーク管理として扱います。これにより、Room 検出、電力グリッド、物流と役割が重なりすぎないようにします。

### 4.1 部屋の「換気基準」と「怠惰の侵食」
- `Room` に `RequiredVentilation`（必要換気量）を定義:
  `RequiredVentilation = Room.tile_count × 0.05`
- 部屋に設置された稼働中（通電中）の `Screaming Vent` および `Scourge Fan` の換気能力の合計がこれを下回る場合、部屋は **「怠惰の侵食（Miasma Infested）」** 状態になります。
- 侵食された部屋にある**本設設備**（今後実装される本設の施設や設備）には、即座に機能停止マーカーが付与され、稼働できなくなります（仮設の `SoulSpa` などの仮設物は免除されているため、侵食下でも動作します）。

換気状態は段階的に扱う案とします。

| 状態 | 条件案 | 効果案 |
|:---|:---|:---|
| `Ventilated` / 換気済み | `provided >= required` | 本設設備が通常稼働可能 |
| `Stagnant` / 停滞 | `provided > 0` かつ `provided < required` | 警告表示。将来は効率低下に使える |
| `MiasmaInfested` / 怠惰侵食 | `provided == 0` または長時間不足 | 本設設備が機能停止 |

### 4.2 流体ネットワーク（排泥システム）
- `LetheIntakeGate`（供給源） ➔ `OssuaryConduit`（配管） ➔ `SludgePurifier`（消費者）の間で、電気グリッドと同様のグラフ接続（流体グリッド）を構築します。
- 配管が川の取水門と繋がっていない、または取水門が停電している場合、`SludgePurifier` は **「断水・泥詰まり」** 状態となり、動作を停止します。
- `SludgeBasin` を導入する場合、供給不足や一時的な停電が起きても短時間だけ本設設備の停止を猶予できます。ただし、バッファが満杯になると関連設備は停止します。

### 4.3 進行段階

| 段階 | 解禁される考え方 | プレイヤー体験 |
|:---|:---|:---|
| 序盤 | 仮設設備と手動水搬送 | 屋外作業・仮設拠点で建設を始める |
| 中盤 | `Room` 成立と小規模換気 | 小部屋を作り、`Screaming Vent` / `Sigh Chimney` で認可する |
| 中盤後半 | 電力付き強制換気 | 大部屋や高負荷設備に `Scourge Fan` を入れる |
| 後半 | 導水・排泥ネットワーク | 川岸から本設区画へ `OssuaryConduit` を伸ばし、停止しない施設網を作る |

## 5. 代替案と比較

| 案 | 採否 | 理由 |
| --- | --- | --- |
| **人間界のエアコン・トイレの直訳導入** | 不採用 | 地獄で働く魂たちに「快適な冷気」や「衛生的なトイレ」を用意するのは世界観（地獄の苦役）と矛盾するため。「建物を破壊する瘴気の換気」と「動作を阻害する罪泥の洗い流し」という建築プロジェクトの要件に翻訳します。 |
| **仮設設備（MudMixerやSoulSpa）も配管接続する** | 不採用 | これらは現場の応急設備であるため。本設の部屋や本設インフラを構築する「本設フェーズ」になって初めて、配管やダクトの本格的な設備工事が必要になるという、建築プロジェクトの進行感・カタルシスを重視します。 |
| **部屋快適度メーターとして実装する** | 不採用 | Soul の快適性管理に見えると地獄の苦役テーマが弱くなるため。あくまで「王の認可」「設備の稼働条件」「建築基準違反」として表現します。 |
