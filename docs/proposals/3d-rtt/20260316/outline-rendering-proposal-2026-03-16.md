## 1. 目的

本提案書はアウトライン生成の**実装仕様ではなく、実装タイミングと前提条件を記録するもの**である。

アウトライン生成はアートスタイルの主要担保であり、以下の未確定事項が解消されるまで具体的な実装設計を行わないことを決定した。早期に設計した場合、PoC 後の全面書き直しリスクがある。

---

## 2. 確定済みの方針

### 2.1 アーキテクチャ

アウトライン生成は以下の2層構造で実装する。この構造は確定している。

```
層A：3Dメッシュ（Bevy Camera3d → RtT）
  全バリアントで同一メッシュを共有
  インスタンシングが完全に効く
  ドローコール：壁全体で LOD 段数分のみ

層B：アウトライン（WGSL Post-process on Camera2d 側）
  RtT テクスチャのエッジ検出 + ランダムノイズで線を揺らす
  画面解像度依存の固定コスト（ドローコールではなくポストプロセス）
  ズームアウト時は無効化する（視認不可なのにコストが同じため）
```

### 2.2 壁メッシュ構成との関係

アウトラインのエッジ検出はジオメトリの稜線・深度差に依存する。外側コーナーの稜線を正しく検出するためには、コーナー専用メッシュが必要である。

```
メッシュ共通（全壁を同一Cuboid）の場合
  外側コーナーの稜線がジオメトリ上に存在しない
  → 深度差ゼロ → エッジ検出されない
  → コーナーが溶けて見える・チープに見える

方法A（コーナー専用メッシュ）の場合
  L字形状のメッシュが1個として存在する
  → 外側コーナーの稜線が深度バッファに正しく現れる
  → エッジ検出が自動的に正しい結果を返す
  → ランダムノイズによる揺らぎもコーナー稜線に正しく乗る
```

**壁メッシュ構成は方法A（コーナー専用メッシュ）を採用する。** これはアウトライン生成の前提条件であり、Phase 3 の GLB 設計時に確定させる。

方法B（パターン単位メッシュ）はアウトライン品質で方法Aと同等だが、インスタンシングが効きにくく spawn コストが高くなるため採用しない。

### 2.3 コスト特性

| 処理 | コスト特性 | 備考 |
| --- | --- | --- |
| RtT オフスクリーン描画 | 2〜4ms（固定） | 描画対象が限定されているため通常のフルシーン RtT より軽い |
| WGSL アウトライン | 2〜4ms（固定） | 画面解像度依存・ズームアウト時は無効化 |
| アウトラインノイズ（WGSL内） | <1ms（解像度比例） | ポストプロセスシェーダー内でフラグメントごとに揺らぎを計算。頂点数には比例しない |
| 合計（GTX 1650） | 6〜13ms | 60fps 上限 16.7ms に対して余裕あり |

アウトラインはズームアウト時（建物が数ピクセルになる状態）に視認不可であるにもかかわらずコストが同じため、ズームアウト時は無効化する。品質設定との連携は以下の通り。

| 品質設定 | アウトライン |
| --- | --- |
| 高 | 有効（ズームイン時） |
| 中 | 有効（ズームイン時のみ） |
| 低 | 無効 |

## 3. 実装設計を保留する理由

アウトライン生成の具体的な実装仕様（エッジ検出カーネル・ノイズ関数・線幅・色・ズーム閾値等）は以下の前提条件が全て揃うまで設計しない。

### 前提条件1：アートスタイル受入基準の確定（P0）

アウトラインの線幅・揺らぎ量・色は `world_lore.md` の記述を元にした受入基準として文書化される必要がある。基準がなければ PoC の合格判断ができず、シェーダーのパラメータ調整に終わりがなくなる。

### 前提条件2：壁メッシュ構成の確定（P0）

方法Aのコーナー専用メッシュが実際に取り込まれるまで、エッジ検出の動作を検証できない。メッシュ構成が変わるとエッジ検出が拾う稜線の数と位置が変わる。

### 前提条件3：断面キャップ方針の確定（P1）

セクションビューで切断面にキャップを描画する場合（SectionMaterial 提案書 セクション8 参照）、アウトラインのエッジ検出とキャップ描画がレンダーパスの順序で干渉しないよう設計する必要がある。キャップ方針が未確定のままアウトラインのレンダーパスを設計すると後から順序の変更が必要になる。

---

## 4. 実装タイミング

```
Phase 3 GLB 取込 PoC
  ↓
アートスタイル受入基準の確定          ← 前提条件1 解消
壁メッシュ構成の確定（方法A）         ← 前提条件2 解消
断面キャップ方針の確定                ← 前提条件3 解消
  ↓
アウトライン生成の実装設計・着手       ← このタイミングで初めて設計する
```

前提条件1〜3 は相互に独立しており、全て Phase 3 の GLB 取込 PoC と連動している。PoC で実際の見た目を確認してから設計することで、シェーダーパラメータの大幅な手戻りを防ぐ。

---

## 5. 実装着手時に設計すべき項目

前提条件が揃った時点で以下を設計する。現時点では設計しない。

- エッジ検出カーネルの選択（Sobel / ラプラシアン / 深度差分）
- ランダムノイズ関数の選択（時間ベース揺らぎ / シードベース固定）
- 線幅・色・アルファの仕様
- ズームアウト無効化の閾値（Camera2d の scale 値）
- 断面キャップとのレンダーパス順序
- 品質設定ごとのパラメータ差分
